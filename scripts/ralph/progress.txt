## 2026-04-07 - F2-021
- Implemented quest system schema: `quests` table (templates) + `questProgress` table (per-user tracking)
- Created migration API route at `src/app/api/migrate-quests/route.ts` with 13 seeded quest templates (8 daily, 5 weekly)
- Created server actions in `src/app/actions/questActions.ts`:
  - `generateDailyQuests(userId)` — picks 3 random daily + 2 weekly quests per day/week (idempotent)
  - `checkQuestProgress(userId)` — measures real metrics (calls_made, appointments_set, leads_contacted) against targets
  - `completeQuest(userId, progressId)` — awards XP + coins with level-up logic
  - `getUserQuests(userId)` — fetches active daily/weekly quests for UI
- Metrics measured from `callLogs` (calls, contacts) and `leads.appointmentCreatedAt` (appointments)
- All dates scoped to Europe/Rome timezone
- File modificati: `src/db/schema.ts`, `src/app/api/migrate-quests/route.ts`, `src/app/actions/questActions.ts`
- **Learnings per iterazioni future:**
  - Quest templates use variable selection (shuffle + slice) per gamification-loops best practice on variable rewards
  - Daily scope format: `YYYY-MM-DD`, weekly scope format: `YYYY-Wnn`
  - `measureMetric()` helper is reusable for future achievement/streak systems
  - Migration uses `ON CONFLICT (id) DO NOTHING` for idempotent seeding
