## Codebase Patterns
- Optimistic locking: ogni UPDATE su leads DEVE includere `eq(leads.version, lead.version)` nel WHERE + version increment + check `.returning()` length
- Timezone: usare `toLocaleDateString('en-CA', { timeZone: 'Europe/Rome' })` per date stringa
- Timezone client: `toLocaleString('it-IT', { timeZone: 'Europe/Rome' })` per display
- Google Calendar: factory `createOAuth2Client()` per ogni operazione, mai singleton
- Server Actions ritornano `{ success: boolean, error?: string }`
- Tailwind v4: custom theme via @theme in globals.css, colori brand: brand-orange (#FFBE82), brand-charcoal (#272626)
- Gamification XP formula: `100 * (Level ^ 1.5)` per target XP, 5 stadi evoluzione Fenice
- `assignedToId` e' immutabile dopo import CSV — mai modificarlo
- DATI REALI: NON toccare lead GDO 114, appuntamenti Conferme, account esistenti
- Streak multiplier: getStreakMultiplier() in `src/lib/streakUtils.ts` — pure function, importable ovunque
- `'use server'` files: TUTTE le export devono essere async. Utility pure functions vanno in `src/lib/`
- Design system premium gia' implementato in Run 1 (F2-014): usare i nuovi token/colori per coerenza
- ConfermeDrawer: usare `salespersonUserId` (UUID) per init, mai `salespersonAssigned` (display name)
- FreeBusy check: SEMPRE prima di db.update(), mai dopo
- Sidebar e Topbar gia' redesignate in Run 1: seguire stesso stile per nuovi componenti
- Achievement system: `checkAchievements(userId)` in `achievementActions.ts` — call after any action that changes metrics (quest complete, call logged, appointment set, level up)
